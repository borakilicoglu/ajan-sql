# Security Policy

## Supported Versions

The latest published release is the supported version.

## Reporting a Vulnerability

If you discover a security issue, please report it privately before opening a public issue.

Use one of these channels:

- GitHub Security Advisories, if enabled for the repository
- direct contact with the maintainer

Please include:

- a clear description of the issue
- reproduction steps
- expected impact
- any suggested mitigation

## Scope

This project is designed around readonly database access and MCP tool safety. Reports involving query guard bypass, unsafe SQL execution, schema exposure issues, or credential handling are especially important.
